# 酒店电竞化改造项目投资评估 - JSON数据模板

## 📋 标准JSON模板（复制使用）

```json
{
  "geographic_location": "",
  "core_customer_flow": {
    "企业年轻员工": "",
    "高校学生": "",
    "商旅与参会客群": ""
  },
  "competitive_pattern": {
    "直接竞品": "",
    "潜在竞品": "",
    "替代娱乐": ""
  },
  "esports_venue_distribution": {
    "1km以内": 0,
    "1-2km": 0,
    "2-3km": 0,
    "备注": ""
  },
  "esports_hotel_distribution": [],
  "business_hotel_distribution": []
}
```

---

## 📖 字段详细说明

### 1. geographic_location（地理位置）
**类型**: `string`（字符串）  
**说明**: 项目所在地理位置的详细描述  
**应包含**: 
- 核心地标（如：阿里巴巴、杭师大等）
- 商圈信息（如：西溪银泰城）
- 交通枢纽（如：地铁站、高铁站）
- 产业集聚区信息

**示例**:
```json
"geographic_location": "位于未来科技城核心区，毗邻阿里巴巴西溪园区、杭师大仓前校区，西溪银泰城商圈、杭州西站交通枢纽环绕，地铁5号线直达"
```

---

### 2. core_customer_flow（核心客流）
**类型**: `object`（对象）  
**必填字段**: 三个客群类型

#### 2.1 企业年轻员工
**字段名**: `"企业年轻员工"`  
**类型**: `string`  
**说明**: 周边企业员工客群分析  
**应包含**: 企业名称、员工数量、年龄段、消费能力

**示例**:
```json
"企业年轻员工": "周边阿里巴巴、网易、字节跳动等大厂，互联网从业者约5万人，年龄25-35岁，消费能力强，电竞需求旺盛"
```

#### 2.2 高校学生
**字段名**: `"高校学生"`  
**类型**: `string`  
**说明**: 周边高校学生客群分析  
**应包含**: 学校名称、距离、学生数量、消费特点

**示例**:
```json
"高校学生": "杭师大仓前校区距离2公里，学生总数约3万人，电竞社团活跃，周末娱乐需求大"
```

#### 2.3 商旅与参会客群
**字段名**: `"商旅与参会客群"`  
**类型**: `string`  
**说明**: 商务出差和会展参会人员分析  
**应包含**: 商务区、会展中心距离、客流量

**示例**:
```json
"商旅与参会客群": "西湖国际博览中心1公里，商务出差、会展参会人员密集，日均流量超1000人次"
```

---

### 3. competitive_pattern（竞争格局）
**类型**: `object`（对象）  
**必填字段**: 三个竞争维度

#### 3.1 直接竞品
**字段名**: `"直接竞品"`  
**类型**: `string`  
**说明**: 直接竞争对手（电竞酒店）分析  
**应包含**: 数量、分布、竞争强度

**示例**:
```json
"直接竞品": "3km范围内无电竞酒店，市场完全空白"
```
或
```json
"直接竞品": "3km内有2家电竞酒店，竞争较激烈"
```

#### 3.2 潜在竞品
**字段名**: `"潜在竞品"`  
**类型**: `string`  
**说明**: 潜在竞争对手（传统网咖等）分析  
**应包含**: 类型、数量、竞争力评估

**示例**:
```json
"潜在竞品": "2家传统网咖，装修陈旧，设备老化，客户体验差"
```

#### 3.3 替代娱乐
**字段名**: `"替代娱乐"`  
**类型**: `string`  
**说明**: 替代性娱乐设施（电影院、KTV等）分析  
**应包含**: 类型、数量、与电竞客群重叠度

**示例**:
```json
"替代娱乐": "电影院2家，KTV 3家，但与电竞客群重叠度低"
```

---

### 4. esports_venue_distribution（电竞馆分布）
**类型**: `object`（对象）  
**字段说明**:

#### 4.1 1km以内
**字段名**: `"1km以内"`  
**类型**: `number`（整数）  
**说明**: 1公里范围内的电竞馆数量

#### 4.2 1-2km
**字段名**: `"1-2km"`  
**类型**: `number`（整数）  
**说明**: 1-2公里范围内的电竞馆数量

#### 4.3 2-3km
**字段名**: `"2-3km"`  
**类型**: `number`（整数）  
**说明**: 2-3公里范围内的电竞馆数量

#### 4.4 备注
**字段名**: `"备注"`  
**类型**: `string`（可选）  
**说明**: 电竞馆分布的补充说明

**示例**:
```json
"esports_venue_distribution": {
  "1km以内": 5,
  "1-2km": 8,
  "2-3km": 3,
  "备注": "周边电竞馆总数16家，市场需求旺盛但不饱和"
}
```

---

### 5. esports_hotel_distribution（电竞酒店分布）
**类型**: `array`（数组）  
**元素格式**: 每个电竞酒店包含三个字段

#### 元素结构:
- **name** (`string`): 酒店名称
- **distance** (`number`): 距离项目的公里数（可以是小数）
- **rooms** (`number`): 电竞房间数量（整数）

**示例**:
```json
"esports_hotel_distribution": [
  {
    "name": "XX电竞酒店",
    "distance": 1.2,
    "rooms": 50
  },
  {
    "name": "YY电竞主题酒店",
    "distance": 2.8,
    "rooms": 30
  }
]
```

**如果没有电竞酒店**:
```json
"esports_hotel_distribution": []
```

---

### 6. business_hotel_distribution（商务酒店分布）
**类型**: `array`（数组）  
**元素格式**: 每个商务酒店包含三个字段

#### 元素结构:
- **name** (`string`): 酒店名称
- **distance** (`number`): 距离项目的公里数（可以是小数）
- **rooms** (`number`): 总房间数量（整数）

**示例**:
```json
"business_hotel_distribution": [
  {
    "name": "如家酒店",
    "distance": 0.8,
    "rooms": 80
  },
  {
    "name": "汉庭酒店",
    "distance": 1.5,
    "rooms": 100
  },
  {
    "name": "全季酒店",
    "distance": 2.0,
    "rooms": 120
  }
]
```

## 🤖 给AI的指令模板

当您有项目数据需要分析时，可以使用以下指令让AI帮您转换：

### 指令模板：

```
请将以下酒店电竞化改造项目信息转换为JSON格式，严格按照以下模板结构：

【在这里粘贴您的原始数据】

JSON模板结构：
{
  "geographic_location": "地理位置描述（字符串）",
  "core_customer_flow": {
    "企业年轻员工": "企业客群描述（字符串）",
    "高校学生": "学生客群描述（字符串）",
    "商旅与参会客群": "商旅客群描述（字符串）"
  },
  "competitive_pattern": {
    "直接竞品": "直接竞争对手描述（字符串）",
    "潜在竞品": "潜在竞争对手描述（字符串）",
    "替代娱乐": "替代娱乐描述（字符串）"
  },
  "esports_venue_distribution": {
    "1km以内": 数字,
    "1-2km": 数字,
    "2-3km": 数字,
    "备注": "备注信息（字符串，可选）"
  },
  "esports_hotel_distribution": [
    {
      "name": "酒店名称",
      "distance": 距离数字,
      "rooms": 房间数数字
    }
  ],
  "business_hotel_distribution": [
    {
      "name": "酒店名称",
      "distance": 距离数字,
      "rooms": 房间数数字
    }
  ]
}

要求：
1. 所有字符串用双引号
2. 数字不加引号
3. 中文字段名保持不变（如"企业年轻员工"）
4. 如果某个酒店列表为空，使用空数组 []
5. 距离可以是小数（如1.2），房间数必须是整数
6. 返回完整可用的JSON，不要有注释
```

## ✅ JSON验证清单

转换完成后，请检查：

- [ ] 所有字符串都用**双引号** `""`
- [ ] 数字**不加引号**（如 `5` 不是 `"5"`）
- [ ] 中文字段名**完全匹配**（"企业年轻员工"、"高校学生"等）
- [ ] 数组元素用**逗号**分隔
- [ ] 最后一个元素**不加逗号**
- [ ] 所有括号正确配对 `{}` 和 `[]`
- [ ] 距离可以是**小数**（如 1.2）
- [ ] 房间数必须是**整数**（如 80）
