## 一、四象限评分模型总览

```
                    ┌──────────────────────────────────────────────┐
                    │                                              │
      回 报          │   Return（蓝）   ──────────   Yield（绿）    │   收 益
      Return 30%    │   地理位置 × 0.4                PCF × 0.5    │   Yield 25%
                    │   核心客流 × 0.6                频率 × 0.5    │
                    │                                              │
                    │────────────────────────────────────────────  │
                    │                                              │
      风 险          │   Risk（红）     ──────────   Control（紫）  │   管 控
      Risk 25%      │   电竞馆密度 × 0.4              商务参照 × 0.4 │   Control 20%
                    │   电竞酒店密度 × 0.6             差异化 × 0.6  │
                    │                                              │
                    └──────────────────────────────────────────────┘
```

### 综合得分公式

```
综合得分 = Return × 0.30 + Yield × 0.25 + Risk × 0.25 + Control × 0.20
```

### 投资价值等级

| 综合得分 | 价值等级 | 建议 | 颜色标识 |
|---------|---------|------|---------|
| ≥ 8.5 | 高投资价值 | 强烈推荐投资 | 绿色 `#10b981` |
| 7.0 – 8.4 | 较高投资价值 | 推荐投资 | 蓝色 `#3b82f6` |
| 5.5 – 6.9 | 中等投资价值 | 谨慎投资，建议优化方案 | 黄色 `#f59e0b` |
| < 5.5 | 投资价值偏低 | 不建议投资，风险较高 | 红色 `#ef4444` |

## 二、八维雷达图轴详解

### 🔵 象限一：回报（Return）— 权重 30%

#### 维度 1：回报强度 `geographic_location`

| 属性 | 值 |
|------|----|
| 雷达轴标签 | 回报强度 |
| 颜色 | `#3b82f6`（蓝色） |
| 权重 | 回报象限 × 40% |
| 满分 | 10 分 |

**评分逻辑（Python `calculate_geographic_score`）**：

```
基础分 = 5.0

高价值关键词（每个 +1.5）：阿里巴巴、杭师大、未来科技城、核心区、商圈、CBD、高新区
中等关键词（每个 +1.0）：产业园、大学城、地铁站、商业区、交通枢纽
低价值关键词（每个 -2.0）：郊区、偏远

最终分 = clamp(基础分 + 加分 - 扣分, 0, 10)
```

**输入字段**：`geographic_location`（文本描述，建议 ≥ 20 字）

**得分解读**：
- 9–10：CBD核心 / 大型科技园区 / 高校密集区
- 7–8：有地铁站 / 商业区
- 5–6：普通住宅区或郊区边缘
- < 5：郊区或偏远地带

---

#### 维度 2：回报质量 `core_customer_flow`

| 属性 | 值 |
|------|----|
| 雷达轴标签 | 回报质量 |
| 颜色 | `#60a5fa`（浅蓝） |
| 权重 | 回报象限 × 60% |
| 满分 | 10 分 |

**评分逻辑（Python `calculate_customer_flow_score`）**：

三类客群加权平均：

| 客群类型 | 权重 | 字段 key |
|---------|------|---------|
| 企业年轻员工 | 40% | `core_customer_flow.企业年轻员工` |
| 高校学生 | 35% | `core_customer_flow.高校学生` |
| 商旅与参会客群 | 25% | `core_customer_flow.商旅与参会客群` |

各客群子得分规则：

| 描述长度 | 基础子分 |
|---------|---------|
| < 20 字 | 0 |
| 20–49 字 | 6 |
| 50–99 字 | 8 |
| ≥ 100 字 | 9 |

关键词加分（每个 +0.5）：`密集`、`活跃`、`需求旺盛`、`消费能力强`、`目标客群`

**得分解读**：
- 9–10：三类客群均有详细描述，且涵盖密集活跃关键词
- 7–8：主要客群有较好覆盖
- < 5：描述过于简略或客群稀少

### 🟢 象限二：收益（Yield）— 权重 25%

> 对接「滴灌通」投资模型的核心财务指标

#### 维度 3：利润空间 `pcf_yield`

| 属性 | 值 |
|------|----|
| 雷达轴标签 | 利润空间 |
| 颜色 | `#10b981`（绿色） |
| 权重 | 收益象限 × 50% |
| 满分 | 10 分 |

**财务模型公式（滴灌通体系）**：

```
RevPAR = ADR × OCC × (1 - OTA比例)
PCF(日) = 房间数 × RevPAR × 分成比例
PCF(月) = PCF(日) × 30

ADR  = 平均每日房价（元/晚）
OCC  = 出租率（%）
OTA  = OTA渠道占比（%）
分成比例 = 投资方收入分成（%，通常 8%–15%）
```

**PCF 日现金流评分映射**：

| PCF 日现金流（元/天） | 得分 |
|-----------------|------|
| ≥ 3,000 | 10.0 |
| 2,000 – 2,999 | 8.5 |
| 1,000 – 1,999 | 7.0 |
| 500 – 999 | 5.0 |
| 200 – 499 | 3.0 |
| < 200 | 1.5 |

**输入字段**：`pcf_daily`（数字，由前端投资计算器自动传入）

**相关计算字段**：

| 字段 | 说明 |
|------|------|
| `adr` | 平均每日房价（元/晚），默认 300 |
| `roomCount` | 房间数量 |
| `occupancyRate` | 出租率（%） |
| `otaRate` | OTA 渠道占比（%） |
| `profitShareRate` | 分成比例（%） |
| `pcf_daily` | 日现金流（自动计算） |
| `pcf_monthly` | 月现金流（= pcf_daily × 30） |

**衍生指标（用于投资回报分析）**：

| 指标 | 公式 | 说明 |
|------|------|------|
| YITO 期（天） | `总投入 / (PCF日 - 总投入×日收益率)` | 回本周期 |
| YITO 期（月） | YITO天 / 30 | |
| ROI | `目标回收 / 总投入` | 投资回报率 |
| MIRR（年化） | 使用 MIRR 算法，再投资率 = 预期回报率 | 修正内部收益率 |

#### 维度 4：自动打款频率 `frequency`

| 属性 | 值 |
|------|----|
| 雷达轴标签 | 自动打款频率 |
| 颜色 | `#34d399`（浅绿） |
| 权重 | 收益象限 × 50% |
| 满分 | 10 分 |

**评分映射（Python `calculate_frequency_score`）**：

| 分账频率 `irr_frequency` | 得分 | 含义 |
|------------------------|------|------|
| `daily`（日分账） | 10.0 | 资金回流最快，流动性最优 |
| `weekly`（周分账） | 7.0 | 良好 |
| `biweekly`（双周分账） | 5.0 | 一般 |
| 其他 / 未填 | 3.0 | 偏低 |

**输入字段**：`irr_frequency`（前端 select，选项：daily / weekly / biweekly）

**业务含义**：频率越高 → 投资方现金流回收越顺畅 → 资金安全性越高

### 🔴 象限三：风险（Risk）— 权重 25%

> **注意**：本象限分数越高 = 风险越低（雷达图面积越大越好的视觉一致性设计）

#### 维度 5：波动可控性 `esports_venue`

| 属性 | 值 |
|------|----|
| 雷达轴标签 | 波动可控性 |
| 颜色 | `#ef4444`（红色） |
| 权重 | 风险象限 × 40% |
| 满分 | 10 分 |

**评分逻辑（Python `calculate_esports_venue_score`）**：

```
总数 = 1km以内 + 1-2km + 2-3km
```

| 3km内电竞馆总数 | 得分 | 风险解读 |
|------------|------|---------|
| ≤ 15 家 | 9.5 | 竞争适中，市场健康 |
| 16 – 20 家 | 7.5 | 竞争较活跃，需差异化 |
| 21 – 25 家 | 6.0 | 竞争较激烈，关注客流分流 |
| 26 – 30 家 | 4.0 | 竞争激烈，风险较高 |
| > 30 家 | 2.0 | 市场过度饱和，高风险 |

**输入字段**：

| 字段 | 含义 |
|------|------|
| `esports_venue_distribution.1km以内` | 1km内电竞馆数量（整数） |
| `esports_venue_distribution.1-2km` | 1–2km内电竞馆数量 |
| `esports_venue_distribution.2-3km` | 2–3km内电竞馆数量 |
| `esports_venue_distribution.备注` | 文字备注（可选） |

#### 维度 6：生命周期可见性 `esports_hotel`

| 属性 | 值 |
|------|----|
| 雷达轴标签 | 生命周期可见性 |
| 颜色 | `#f87171`（浅红） |
| 权重 | 风险象限 × 60% |
| 满分 | 10 分 |

**评分逻辑（Python `calculate_esports_hotel_score`）**：

无竞争电竞酒店 → 直接得满分 10.0

有竞争时，计算加权竞争强度分：

```
竞争强度分 = Σ (房间数 × 距离权重)

距离权重：
  ≤ 1km：× 0.8（影响最大）
  1–2km：× 0.5
  2–3km：× 0.3
```

| 竞争强度分 | 得分 |
|----------|------|
| ≤ 30 | 10.0（市场空白大） |
| 31 – 60 | 8.0 |
| 61 – 100 | 6.0 |
| > 100 | 4.0（竞争极激烈） |

**输入字段**（数组 `esports_hotel_distribution`，每项含）：

| 子字段 | 类型 | 含义 |
|-------|------|------|
| `name` | string | 电竞酒店名称（必填） |
| `distance` | float | 距离（km，≥ 0） |
| `rooms` | int | 客房数量 |
| `price_range` | string | 价格区间（元/晚，如 "199–299"） |
| `grade` | string | 等级（如 "经济型"、"三星级"） |

| 权重 | 管控象限 × 60% |
| 满分 | 10 分 |


## 四、评分计算完整流程

```
输入 JSON
    │
    ▼
【子维度计算（8个）】
    ├── geographic_location  → calculate_geographic_score()
    ├── core_customer_flow   → calculate_customer_flow_score()
    ├── pcf_yield            → calculate_pcf_score()         ← 依赖 pcf_daily
    ├── frequency            → calculate_frequency_score()   ← 依赖 irr_frequency
    ├── esports_venue        → calculate_esports_venue_score()
    ├── esports_hotel        → calculate_esports_hotel_score()
    ├── business_hotel       → calculate_business_hotel_score()
    └── differentiation      → calculate_differentiation_score()
    │
    ▼
【四象限聚合】
    ├── Return  = geo×0.4  + customer×0.6       → 上限 10分
    ├── Yield   = pcf×0.5  + frequency×0.5      → 上限 10分
    ├── Risk    = venue×0.4 + hotel×0.6          → 上限 10分（分高=风险低）
    └── Control = business×0.4 + diff×0.6        → 上限 10分
    │
    ▼
【综合得分】
    综合 = Return×30% + Yield×25% + Risk×25% + Control×20%
    │
    ▼
【价值等级判定】
    ≥8.5 → 高投资价值（强烈推荐）
    ≥7.0 → 较高投资价值（推荐）
    ≥5.5 → 中等投资价值（谨慎）
    <5.5 → 投资价值偏低（不建议）
    │
    ▼
【输出结构】
    {
      quadrant_scores: {return, yield, risk, control},
      comprehensive_score,
      value_level,
      recommendation,
      dimension_scores: {8个子维度得分},
      conclusion: {strengths, risks, suggestions}
    }
```

## 五、结论生成规则

系统根据四象限分自动生成三类结论：

### 优势（Strengths）

| 触发条件 | 输出文本 |
|---------|---------|
| Return ≥ 8 | 回报潜力强，地段与客群匹配度高 |
| Yield ≥ 7 | 收益质量优，PCF现金流充裕且分账顺畅 |
| Risk ≥ 8 | 竞争风险低，市场空白明显，先发优势突出 |
| Control ≥ 7 | 管控能力强，差异化护城河清晰 |

### 风险警示（Risks）

| 触发条件 | 输出文本 |
|---------|---------|
| Return < 6 | 回报维度偏弱，地理位置或客群覆盖不足 |
| Yield < 5 | 收益维度偏低，PCF现金流或分账频率需优化 |
| Risk < 6 | 市场竞争较激烈，电竞馆或同类酒店密集 |
| Control < 5 | 管控能力薄弱，差异化优势不明显 |

### 优化建议（Suggestions）

| 触发条件 | 建议 |
|---------|------|
| geo < 7 | 建议选择靠近高校、产业园或核心商圈的位置 |
| pcf < 5 | 建议提升房间数量或ADR，扩大PCF日现金流 |
| frequency < 7 | 建议与投资方协商更高频的分账周期（如日分账） |
| hotel < 6 | 建议打造差异化主题房型或服务体验 |
| diff < 6 | 建议强化产品差异化，构建电竞IP/场景/会员壁垒 |

## 六、雷达图可视化说明

- **图表类型**：Chart.js Radar Chart
- **轴数量**：8 轴（每象限 2 个子轴）
- **轴范围**：0 – 10，步长 2
- **轴顺序**（顺时针）：
  1. 回报强度（Return 象限）
  2. 回报质量（Return 象限）
  3. 利润空间（Yield 象限）
  4. 打款频率（Yield 象限）
  5. 波动可控性（Risk 象限）
  6. 生命周期可见性（Risk 象限）
  7. 市场参照成熟度（Control 象限）
  8. 差异化管控力（Control 象限）

- **颜色方案**：
  - Return 区域：蓝色系（`#3b82f6` / `#60a5fa`）
  - Yield 区域：绿色系（`#10b981` / `#34d399`）
  - Risk 区域：红色系（`#ef4444` / `#f87171`）
  - Control 区域：紫色系（`#8b5cf6` / `#a78bfa`）

- **进度条颜色规则**：
  - 分数 ≥ 8 → 绿色（优秀）
  - 分数 ≥ 6 → 蓝色（良好）
  - 分数 ≥ 4 → 橙色（一般）
  - 分数 < 4 → 红色（偏弱）

## 七、关键业务规则摘要

### 最优项目画像（雷达图面积最大化）

```
地理位置：核心商圈 / 高新区 / CBD + 地铁直达         → geo ≥ 8
客群描述：三类客群均有 ≥ 100 字详细描述 + 积极关键词    → customer ≥ 8
PCF日现金流：≥ 2000 元/天（100间×300ADR×75%OCC×9%分成）→ pcf ≥ 8.5
分账频率：日分账（daily）                              → frequency = 10
电竞馆密度：3km内 ≤ 15 家                              → venue = 9.5
电竞酒店竞争：竞争强度分 ≤ 30（无或极少竞品）            → hotel ≥ 10
商务酒店：3km内总房间 100–300 间                        → business = 9
差异化：文本含 "空白" / "首家" / "独家" 等关键词          → diff ≥ 8
```

### 红线预警（综合分 < 5.5）

```
- 3km内电竞酒店竞争强度分 > 100（多家大型竞品紧邻）
- PCF日现金流 < 200 元/天（项目财务不可行）
- geographic_location 含 "郊区" 或 "偏远" 关键词
- core_customer_flow 三类客群描述均 < 20 字
```

## 九、典型评估案例

### 案例：某市核心商圈电竞改造项目

**输入摘要**：
- 地理：核心商圈 + 大型科技园区 + 地铁 → geo = 7.0
- 客群：详细描述三类 → customer = 6.0
- PCF日：约 900 元/天（100间×300×75%×4%） → pcf_score = 5.0（注：需提高分成比例）
- 分账：daily → frequency = 10
- 电竞馆：1km内5家 + 1-2km内8家 + 2-3km内3家 = 16家 → venue = 7.5
- 电竞酒店：2家（竞争强度 ≈ 35） → hotel = 8.0
- 商务酒店：3家共280间 → business = 9.0
- 差异化：含"差异化"、"空白" → diff = 7.0

**计算过程**：
```
Return  = 7.0×0.4 + 6.0×0.6 = 2.80 + 3.60 = 6.40
Yield   = 5.0×0.5 + 10×0.5  = 2.50 + 5.00 = 7.50
Risk    = 7.5×0.4 + 8.0×0.6 = 3.00 + 4.80 = 7.80
Control = 9.0×0.4 + 7.0×0.6 = 3.60 + 4.20 = 7.80

综合 = 6.40×0.30 + 7.50×0.25 + 7.80×0.25 + 7.80×0.20
     = 1.920 + 1.875 + 1.950 + 1.560
     = 7.31 → 「较高投资价值，推荐投资」
```
