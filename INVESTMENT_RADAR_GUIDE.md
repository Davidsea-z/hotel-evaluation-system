## 一、赛道背景

**电竞酒店**：在传统酒店客房内配置高性能电竞 PC（RTX 50 系显卡）、外设及电竞场景，面向 18–35 岁年轻用户群体，以电竞娱乐为核心差异化卖点的主题酒店。

**核心逻辑**：  
- 传统酒店出租率低（50–65%）→ 电竞改造后出租率提升至 85–95%  
- ADR（日均房价）从 160–200 元提升至 240–350 元  
- 增量收益通过「滴灌通联营模型」实现投资方按日/周/双周自动分账

**行业数据参考**（2020–2025）：

|------|--------------|------------|
| 2020 | 238.5 | 4.44 |
| 2021 | 254.3 | 4.59 |
| 2022 | 267.8 | 4.72 |
| 2023 | 275.7 | 4.82 |
| 2024 | 283.9 | 4.89 |

## 二、投资模型（滴灌通体系）

**滴灌通联营模型**：投资方出资改造电竞客房，与酒店方按约定比例分配收入，分账自动化、周期化（日/周/双周），无需介入日常运营。

### 2.1 核心概念定义

| 术语 | 全称 | 含义 |
|------|------|------|
| ADR | Average Daily Rate | 平均每日房价（元/晚）|
| OCC | Occupancy Rate | 出租率（%） |
| OTA | Online Travel Agency | OTA 渠道（携程/美团等），占比影响实际可分成收入 |
| RevPAR | Revenue Per Available Room | 每间可用客房收入 |
| PCF | Profit Cash Flow | 投资方日现金流（联营分成） |
| YITO | Yield to Investor Term Objective | 联营期限，即投资方完成本金回收所需时间 |
| MIRR | Modified Internal Rate of Return | 修正内部收益率（考虑再投资收益） |
| ROI | Return on Investment | 投资回报倍数 |

### 2.2 联营分成结构

```
酒店总收入
    │
    ├── OTA 渠道佣金（约 9%，不进入分成基数）
    │
    └── 净收入 = ADR × OCC × (1 - OTA%)
                │
                ├── 酒店方保留（1 - 分成比例）
                │
                └── 投资方分成 → PCF（日现金流）
```

**分成比例行业参考**：通常 8%–15%，具体按合同约定

## 四、核心财务公式推导链

### 4.1 RevPAR（每间可用客房收入）

```
RevPAR = ADR × OCC × (1 - OTA比例)

示例：
  ADR = 300 元，OCC = 85%，OTA = 9%
  RevPAR = 300 × 0.85 × (1 - 0.09)
         = 300 × 0.85 × 0.91
         = 232.05 元/间/天
```

> **意义**：RevPAR 是剔除 OTA 渠道成本后，真正可用于分成的每间收入基数。

### 4.2 PCF（日现金流）

```
PCF(日) = 房间数 × RevPAR × 分成比例

示例：
  房间数 = 100，RevPAR = 232.05，分成 = 10%
  PCF(日) = 100 × 232.05 × 0.10 = 2,320.5 元/天

PCF(月) = PCF(日) × 30 = 69,615 元/月
```

> **意义**：PCF 是投资方每天实际收到的现金流，是 YITO/MIRR 计算的基础。

### 4.3 YITO（联营期限/回本周期）

**推导原理**：投资方在联营期内每天收到 PCF，同时总投资本金在以目标年收益率「增值」，YITO 是两者相等时的天数。

```
YITO(天) = 总投资(元) / (PCF(日) - 总投资(元) × 日收益率)

其中：
  日收益率 = 年收益率 / 365
  总投资(元) = totalRenovationCost × 10000（万元转元）

示例：
  总投资 = 50万 = 500,000 元
  PCF(日) = 2,320.5 元
  日收益率 = 18% / 365 = 0.0493%
  分母 = 2,320.5 - 500,000 × 0.000493 = 2,320.5 - 246.6 = 2,073.9
  YITO = 500,000 / 2,073.9 ≈ 241 天（约 8 个月）

YITO(月) = YITO(天) / 30

前提条件：PCF(日) > 总投资 × 日收益率，否则 YITO 无解（项目不可行）
```

### 4.4 目标回收总额

```
目标回收总额(元) = PCF(日) × YITO(天)

示例：2,320.5 × 241 = 559,240.5 元
```

### 4.5 ROI（投资回报倍数）

```
ROI = 目标回收总额 / 总投资(元)

示例：559,240.5 / 500,000 = 1.12 倍

注意：ROI > 1 说明投资方在联营期内实现本金回收+增值
```

### 4.6 完整计算示例

| 参数 | 数值 |
|------|------|
| 房间数 | 100 |
| ADR | 300 元 |
| 出租率 | 85% |
| OTA比例 | 9% |
| 分成比例 | 10% |
| 总投资 | 50 万元 |
| 预期年收益率 | 18% |
| 分账频率 | daily（日分账） |

**→ 计算结果**：

| 指标 | 数值 |
|------|------|
| RevPAR | 232.05 元/间/天 |
| PCF(日) | 2,320.5 元/天 |
| PCF(月) | 69,615 元/月 |
| YITO(天) | 约 241 天 |
| YITO(月) | 约 8.0 个月 |
| ROI | 约 1.12 倍 |
| MIRR（日分账） | 约 18–22%（取决于再投资率） |

## 五、IRR / MIRR 算法说明

### 5.1 为什么使用 MIRR 而非普通 IRR

传统 IRR 假设所有回收现金流都以 IRR 本身的利率再投资，这在现实中不合理（再投资利率通常低于 IRR）。MIRR 允许指定真实再投资利率（系统中使用 `expectedReturn` 即预期年收益率）。

### 5.2 MIRR 计算公式

```
步骤：
1. 按再投资利率，将所有现金流复利滚到项目期末 → 终值 FV
2. MIRR = (FV / 初始投资)^(1/n) - 1
   其中 n = 项目总期限（年） = YITO(天) / 365

代码公式：
  FV = Σ [cashFlow_i × (1 + reinvestmentRate)^(remainingYears_i)]
  remainingYears_i = (maxDays - flow.days) / 365
  MIRR = (FV / totalInvestment)^(1 / maxYears) - 1
```

### 5.3 三种分账频率的现金流构造

| 分账频率 | 现金流构造方式 | 现金流笔数示例（YITO=241天） |
|---------|------------|----------------------|
| `daily`（日分账） | 每天一笔，金额 = PCF(日) | 241 笔 |
| `weekly`（周分账） | 每 7 天一笔，金额 = PCF(日) × 7 | 约 35 笔 |
| `biweekly`（双周分账） | 每 14 天一笔，金额 = PCF(日) × 14 | 约 18 笔 |

> **影响**：分账越频繁，复投时间越早，FV 越大，MIRR 越高。日分账的 MIRR 高于周分账，高于双周分账。

### 5.4 牛顿-拉夫逊迭代法（标准 IRR）

系统也保留了标准 IRR 函数（`calculateIRRByDays`），用于对比验证：

```
初始猜测：irr = 0.1（10%）
迭代公式：irr_new = irr - NPV(irr) / NPV'(irr)
收敛条件：|irr_new - irr| < 1e-8 且 |NPV| < 1e-6
边界保护：irr ∈ [-0.99, 10.0]（防止指数爆炸）
```

## 六、项目评估打分系统（评估器）

> 注意：此系统为「页面内独立评估器」，与后端四象限 API 并行存在，适用于快速项目筛选评估（满分 100 分）。

### 6.1 四维打分结构

| 维度 | 满分 | 权重 |
|------|------|------|
| 财务收益 | 30 | 30% |
| 市场潜力 | 25 | 25% |
| 回本周期 | 25 | 25% |
| 风险控制 | 20 | 20% |

### 6.2 财务收益评分（满分 30）

```
预计月收入 = 改造房间数 × 目标房价 × (目标出租率/100) × 30
预计年收入 = 预计月收入 × 12 / 10000（万元）
预计年利润 = 预计年收入 × 65%（利润率假设）
ROI(%) = 预计年利润 / 改造预算 × 100
```

| ROI 范围 | 得分 |
|---------|------|
| ≥ 200% | 30 |
| 150–199% | 25 |
| 100–149% | 20 |
| 80–99% | 15 |
| < 80% | 10 |

### 6.3 市场潜力评分（满分 25）

基于地理位置评级直接赋分：

| 区域类型 | 得分 | ADR倍增系数 | 出租率加成 |
|---------|------|-----------|---------|
| 核心商圈（core） | 25 | ×1.5 | +35% |
| 城市开发区（development） | 20 | ×1.35 | +30% |
| 产城融合区（integration） | 18 | ×1.25 | +25% |
| 景区周边（scenic） | 22 | ×1.4 | +30% |

### 6.4 回本周期评分（满分 25）

```
回本月数 = 改造预算 / 预计年利润 × 12
```

| 回本月数 | 得分 | 评级 |
|---------|------|------|
| ≤ 6 个月 | 25 | 极快 |
| 7–10 个月 | 20 | 较快 |
| 11–15 个月 | 15 | 中等 |
| 16–24 个月 | 10 | 较慢 |
| > 24 个月 | 5 | 慢 |

### 6.5 风险控制评分（满分 20）

基础分来自改造方案定位（GPU等级）：

| 方案定位 | 基础分 | 单间设备预算 |
|---------|-------|----------|
| 旗舰版（RTX5070+） | 20 | 3.5–5万元/间 |
| 标准版（RTX5060Ti/5070） | 18 | 2.5–3.5万元/间 |
| 经济版（RTX5060） | 15 | 2–2.5万元/间 |

扣分规则：

| 条件 | 扣分 |
|------|------|
| 预算低于方案最低要求 | -3 |
| 预算高于方案最高要求 | -1 |
| 电竞设备占比 60–70% | -2 |
| 电竞设备占比 < 60% | -4 |

### 6.6 综合评级

```
综合分 = 财务 + 市场 + 回本 + 风险
```

| 综合分 | 评级 | 建议 |
|-------|------|------|
| ≥ 85 | 优秀项目 | 强烈推荐投资 |
| 70–84 | 良好项目 | 推荐投资 |
| 55–69 | 中等项目 | 谨慎投资 |
| < 55 | 需改进 | 暂缓或重新评估 |

## 七、四象限雷达图评分模型

> 此模型由后端 `app.py` 计算，通过 `/api/calculate_score` 接口返回，适用于深度市场调研评估。

### 7.1 四象限结构

```
                ┌───────────────────────────────────────────────┐
                │                                               │
  回 报          │  Return（蓝）  ─────────────  Yield（绿）    │  收 益
  Return 30%    │  地理位置 ×0.4               PCF ×0.5        │  Yield 25%
                │  核心客流 ×0.6               分账频率 ×0.5    │
                │                                               │
                │───────────────────────────────────────────────│
                │                                               │
  风 险          │  Risk（红）    ─────────────  Control（紫）  │  管 控
  Risk 25%      │  电竞馆密度 ×0.4             商务参照 ×0.4   │  Control 20%
                │  电竞酒店密度 ×0.6           差异化管控 ×0.6  │
                └───────────────────────────────────────────────┘
```

### 7.2 综合得分公式

```
四象限分计算：
  Return  = geo_score × 0.4  + customer_score × 0.6
  Yield   = pcf_score × 0.5  + frequency_score × 0.5
  Risk    = venue_score × 0.4 + hotel_score × 0.6
  Control = business_score × 0.4 + diff_score × 0.6

综合得分 = Return×0.30 + Yield×0.25 + Risk×0.25 + Control×0.20
```

> ⚠️ **重要**：Risk 象限分数越高 = 风险越低，设计上保持雷达图面积越大越好的视觉一致性。

### 7.3 投资价值等级

| 综合得分 | 价值等级 | 建议 | 颜色 |
|---------|---------|------|------|
| ≥ 8.5 | 高投资价值 | 强烈推荐投资 | `#10b981` 绿 |
| 7.0–8.4 | 较高投资价值 | 推荐投资 | `#3b82f6` 蓝 |
| 5.5–6.9 | 中等投资价值 | 谨慎投资，建议优化 | `#f59e0b` 黄 |
| < 5.5 | 投资价值偏低 | 不建议投资 | `#ef4444` 红 |

### 🔴 维度 6：生命周期可见性 `esports_hotel`（Risk 象限，权重 60%）

无直接竞品 → 满分 10。有竞品时：

```
竞争强度分 = Σ (rooms × 距离权重)
  ≤1km：× 0.8
  1–2km：× 0.5
  2–3km：× 0.3
```

| 竞争强度分 | 得分 |
|----------|------|
| ≤ 30 | 10.0（市场空白大） |
| 31–60 | 8.0 |
| 61–100 | 6.0 |
| > 100 | 4.0（竞争极激烈） |

**输入字段**（数组 `esports_hotel_distribution`）每项含：`name`、`distance`（km）、`rooms`、`price_range`、`grade`

---

### 🟣 维度 7：市场参照成熟度 `business_hotel`（Control 象限，权重 40%）

商务酒店总客房适量时市场最成熟、定价锚点最健康：

| 3km内商务酒店总客房 | 得分 |
|----------------|------|
| 0（无） | 6.0（商务氛围缺失） |
| ≤ 100 间 | 7.0 |
| 101–300 间 | 9.0（最佳：氛围好且竞争有限）✅ |
| 301–500 间 | 8.0 |
| > 500 间 | 6.0（竞争过激） |

---

### 🟣 维度 8：差异化管控力 `differentiation`（Control 象限，权重 60%）

```
分析文本 = 直接竞品描述 + 潜在竞品描述 + 替代娱乐描述
基础分 = 5.0

正向关键词（+1.0/个）：空白、差异化、独家、唯一、首家、无竞品、无直接
负向关键词（-0.8/个）：激烈、饱和、多家、竞争强、红海

最终分 = clamp(基础分 ± 调整, 0, 10)
```

## 十、评分计算完整流程

```
输入 JSON
    │
    ▼
【8个子维度计算】
    ├── geographic_location  → calculate_geographic_score()
    ├── core_customer_flow   → calculate_customer_flow_score()
    ├── pcf_yield            → calculate_pcf_score()         ← 依赖 pcf_daily
    ├── frequency            → calculate_frequency_score()   ← 依赖 irr_frequency
    ├── esports_venue        → calculate_esports_venue_score()
    ├── esports_hotel        → calculate_esports_hotel_score()
    ├── business_hotel       → calculate_business_hotel_score()
    └── differentiation      → calculate_differentiation_score()
    │
    ▼
【四象限聚合】
    Return  = geo×0.4  + customer×0.6
    Yield   = pcf×0.5  + frequency×0.5
    Risk    = venue×0.4 + hotel×0.6
    Control = business×0.4 + diff×0.6
    │
    ▼
【综合得分 = Return×30% + Yield×25% + Risk×25% + Control×20%】
    │
    ▼
【价值等级判定 → 结论生成】
    │
    ▼
【返回结构】
{
  quadrant_scores: { return_score, yield_score, risk_score, control_score },
  comprehensive_score,
  value_level,
  recommendation,
  color,
  dimension_scores: { 8个子维度得分 },
  conclusion: { strengths[], risks[], suggestions[] }
}
```

## 十一、结论自动生成规则

### 优势（Strengths）触发条件

| 条件 | 输出文本 |
|------|---------|
| Return ≥ 8 | 回报潜力强（X分），地段与客群匹配度高 |
| Yield ≥ 7 | 收益质量优（X分），PCF现金流充裕且分账顺畅 |
| Risk ≥ 8 | 竞争风险低（X分），市场空白明显，先发优势突出 |
| Control ≥ 7 | 管控能力强（X分），差异化护城河清晰 |

### 风险警示（Risks）触发条件

| 条件 | 输出文本 |
|------|---------|
| Return < 6 | 回报维度偏弱，地理位置或客群覆盖不足 |
| Yield < 5 | 收益维度偏低，PCF现金流或分账频率需优化 |
| Risk < 6 | 市场竞争较激烈，电竞馆或同类酒店密集 |
| Control < 5 | 管控能力薄弱，差异化优势不明显 |

### 优化建议（Suggestions）触发条件

| 子维度得分 | 建议 |
|----------|------|
| geo < 7 | 建议选择靠近高校、产业园或核心商圈的位置 |
| pcf < 5 | 建议提升房间数量或ADR，扩大PCF日现金流规模 |
| frequency < 7 | 建议与投资方协商更高频的分账周期（如日分账） |
| hotel < 6 | 建议打造差异化主题房型或服务体验 |
| diff < 6 | 建议强化差异化，构建电竞IP/场景/会员体系壁垒 |

## 十三、关键业务规则速查

### 最优项目画像（雷达图面积最大化）

```
地理位置：核心商圈 + 高校/产业园 + 地铁直达     → geo ≥ 8
客群描述：三类客群均有 ≥100字 + 积极关键词       → customer ≥ 8
PCF：    ≥ 2,000 元/天                          → pcf_score = 8.5+
         （参考：100间 × 300ADR × 85%OCC × 9%分成 ÷ 0.91 ≈ 2,530元）
分账：   日分账                                  → frequency = 10
电竞馆：  3km内 ≤ 15家                            → venue = 9.5
电竞酒店：无或极少竞品（竞争强度 ≤ 30）            → hotel ≥ 10
商务酒店：3km内总房间 100–300间                   → business = 9
差异化：  文本含"空白"/"首家"/"独家"等正向词        → diff ≥ 8
```

### 红线预警（综合分 < 5.5）

```
⛔ 3km内电竞酒店竞争强度分 > 100（多家大型紧邻竞品）
⛔ PCF日现金流 < 200元/天（项目财务不可行）
⛔ geographic_location 含"郊区"或"偏远"
⛔ 三类客群描述均 < 20字
⛔ PCF(日) ≤ 总投资 × 日收益率（YITO无解，无法回本）
```
